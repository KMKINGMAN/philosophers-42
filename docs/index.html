The Dining Philosophers

The Dining Philosophers problem is a classic example in computer science used to illustrate synchronization issues and techniques for resolving them. It was originally formulated in 1965 by Edsger Dijkstra as a student exam exercise, and later put in its current form by Tony Hoare.

This project implements a solution to the Dining Philosophers problem using threads and mutexes in C, demonstrating an understanding of concurrent programming concepts.

Key Features

Multi-threaded Simulation

Each philosopher is represented as an independent thread, simulating real-world concurrent processes.

lock
Mutex Management

Demonstrates proper use of mutex locks to prevent race conditions and ensure synchronized access to shared resources.

settings_backup_restore
Deadlock Prevention

Implements strategies to prevent deadlocks where philosophers would wait indefinitely for resources.
